## 📖 Описание проекта

Данная система представляет собой интеллектуальный таск-трекер, который не просто хранит задачи, но и помогает пользователю их планировать. Система использует методы машинного обучения (ML) и обработки естественного языка (NLP) для:

*   **Автоматической декомпозиции** сложных задач на подзадачи.
*   **Персонализированной оценки** времени выполнения (учитывает вашу скорость работы).
*   **Генерации расписаний** с учетом приоритетов и загрузки.
*   **Self-finetuning (Самообучение):** Система анализирует фактическое время выполнения задач и автоматически корректирует свои прогнозы для будущих планирований.

Основная цель — снижение когнитивной нагрузки на пользователя и повышение точности планирования.

## 🛠 Технологический стек

Согласно архитектурному проекту (Раздел 3 ВКР):

*   **Frontend:** React / Next.js, TypeScript, MUI (Material UI).
*   **Backend:** Node.js (Nest.js / Express), TypeScript, JWT Auth.
*   **Database:** PostgreSQL (основное хранилище), TypeORM / Prisma.
*   **ML Service:** Python (FastAPI / Flask), Scikit-learn / PyTorch (для прогнозирования и NLP).
*   **Infrastructure:** Docker, Docker Compose, Nginx (опционально).

## 🚀 Быстрый старт (Рекомендуемый способ)

Для запуска всех компонентов системы рекомендуется использовать **Docker Compose**. Это обеспечит изолированное окружение для БД, Бэкенда и ML-сервиса.

### 1. Пререквизиты

Убедитесь, что у вас установлены:
*   [Node.js](https://nodejs.org/) (версия 18+)
*   [Python](https://www.python.org/) (версия 3.9+)
*   [Docker](https://www.docker.com/) и [Docker Compose](https://docs.docker.com/compose/)
*   Git

### 2. Клонирование репозитория

```bash
git clone <ссылка_на_репозиторий>
cd <имя_папки_проекта>
```

### 3. Настройка переменных окружения

В корне проекта создайте файл `.env` на основе примера `.env.example` (если существует) или создайте вручную:

```bash
# .env
# Database
POSTGRES_USER=postgres
POSTGRES_PASSWORD=secure_password
POSTGRES_DB=task_tracker_db
DATABASE_URL=postgresql://postgres:secure_password@db:5432/task_tracker_db

# Backend
JWT_SECRET=super_secret_jwt_key_change_me
PORT=3000

# ML Service
ML_SERVICE_URL=http://ml_service:8000
```

### 4. Запуск системы

В терминале из корневой папки выполните:

```bash
docker-compose up --build
```

Эта команда:
1.  Поднимет контейнер с **PostgreSQL**.
2.  Запустит **Backend** (Node.js) и применит миграции БД.
3.  Запустит **ML Service** (Python) и загрузит необходимые модели.
4.  Запустит **Frontend** (если настроено в compose) или оставит его для локального запуска (см. ниже).

> **Примечание:** Если вы разрабатываете Frontend локально, удобнее запускать только БД и Бэкенд через Docker, а фронт через `npm run dev`.

## 💻 Локальная разработка (Покомпонентно)

Если вы хотите запускать части проекта отдельно для отладки.

### 1. База данных (PostgreSQL)

Проще всего поднять БД через Docker:

```bash
docker run --name pg-tracker -e POSTGRES_PASSWORD=secure_password -e POSTGRES_DB=task_tracker_db -p 5432:5432 -d postgres
```

### 2. Backend (Node.js)

```bash
cd backend
npm install
# Настройте .env файл в папке backend
npm run start:dev
```
*Бэкенд будет доступен по адресу: `http://localhost:3000`*

### 3. Frontend (React/Next.js)

```bash
cd frontend
npm install
# Настройте .env файл (укажите API_URL=http://localhost:3000)
npm run dev
```
*Фронтенд будет доступен по адресу: `http://localhost:3001` (или 3000, если бэкенд на другом порту)*

### 4. ML Service (Python)

Так как навыки в ML базовые, сервис обернут в простой API.

```bash
cd ml_service
# Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate

# Установите зависимости
pip install -r requirements.txt

# Запустите сервис
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
*ML сервис будет доступен по адресу: `http://localhost:8000`*

## 🧠 Как работает Self-Finetuning (Самообучение)

Согласно разделу 2.1.6 ВКР, система обучается автоматически в фоновом режиме.

1.  **Сбор данных:** Когда вы отмечаете задачу как выполненную, система записывает `плановое время` и `фактическое время`.
2.  **Анализ:** Скрипт (Cron или фоновая задача) вычисляет коэффициент скорости пользователя ($k_{скорости}$).
3.  **Обновление:** При создании новых задач модель использует обновленный коэффициент для более точной оценки.
4.  **Глубокий анализ:** Если ошибка прогноза > 20%, запускается процедура корректировки весов модели (Раздел 2.1.7).

*Вам не нужно запускать обучение вручную. Всё происходит автоматически при использовании системы.*

## 🐛 Troubleshooting (Решение проблем)

1.  **Ошибка подключения к БД:**
    *   Проверьте, запущен ли контейнер PostgreSQL.
    *   Убедитесь, что `DATABASE_URL` в `.env` совпадает с параметрами запуска Docker.
2.  **Ошибка импортов в ML сервисе:**
    *   Убедитесь, что активировано виртуальное окружение Python.
    *   Проверьте версии библиотек в `requirements.txt`.
3.  **CORS ошибки на фронтенде:**
    *   Проверьте настройки CORS в `backend/main.ts` (или аналогичном файле). Должен быть разрешен адрес вашего фронтенда (например, `localhost:3001`).
